PERBAIKAN BUG LIVE PREVIEW

Layout Tidak Boleh Bergeser

Saat modal kemenangan muncul:

Jangan sampai:



Tombol Hint bergeser

Tombol Periksa Jawaban bergeser

Status Bar berubah posisi

Panel berubah ukuran

Gunakan:



position: fixed;

untuk seluruh modal.

Modal harus berupa overlay.

Muncul dan hilangnya modal tidak boleh mengubah layout utama.

Font Size Tidak Mengubah Preview (Level 2)

Saat pemain mengubah:



font-size

preview pada:



HASIL NILAIMU

harus langsung berubah.

Contoh:



font-size: 24px;

menjadi:



font-size: 48px;

ukuran teks harus langsung berubah secara real-time.

Gap Tidak Mengubah Preview (Level 3)

Saat pemain mengubah:



gap

jarak antar elemen harus langsung berubah pada preview.

Perubahan harus real-time.

Gap Tidak Mengubah Preview (Level 4)

Bug yang sama harus diperbaiki.

Nilai:



gap

harus langsung memengaruhi layout preview.

Semua Properti Ukuran Harus Real-Time

Pastikan seluruh properti berikut langsung mengubah preview:



font-size

line-height

letter-spacing

gap

border-radius

width

height

padding

margin

Perubahan input harus langsung mengubah tampilan preview.

Tidak boleh membutuhkan refresh.

Preview Harus Selalu Sinkron

Alur yang wajib terjadi:



Input berubah

↓

State berubah

↓

Preview dirender ulang

↓

Tampilan berubah

Tidak boleh ada kondisi:



Input berubah

State berubah

Preview tetap sama

Preview harus selalu mencerminkan nilai terbaru.

ISOLASI PREVIEW

CSS yang sedang dimainkan tidak boleh mengubah UI aplikasi.

Contoh yang tidak boleh terjadi:



Tombol Periksa Jawaban ikut berubah ukuran

Status Bar ikut berubah

Header ikut berubah

Panel Editor ikut berubah

CSS level hanya boleh memengaruhi:



HASIL NILAIMU

dan



TARGET

Gunakan style isolation atau scoped styling.

BATAS CONTAINER PREVIEW

Jika pemain memasukkan nilai ekstrem:



font-size: 500px;

gap: 300px;

width: 2000px;

preview tidak boleh merusak layout aplikasi.

Container HASIL NILAIMU harus memiliki perilaku:



overflow: auto;

max-width: 100%;

max-height: 100%;

atau pendekatan setara.

Jika preview terlalu besar:



Munculkan scrollbar

Jangan merusak layout aplikasi
